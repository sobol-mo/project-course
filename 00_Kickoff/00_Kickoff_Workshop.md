## Блок 1: Розподіл ролей (10 хв)

### Інструкція для викладача

Запитайте студентів:

1. **«Хто хоче бути Project Manager?»** — комунікабельний, організований. Не обов'язково технічно найсильніший.
2. **«Хто хоче бути Architect?»** — системно мислить, готовий приймати рішення і пояснювати їх. Зазвичай технічно сильний.
3. Далі — **двоє Developers** (хочуть писати код).
4. **QA** — уважний, методичний, не боїться ламати чужий код тестами.
5. **DevOps** — цікавиться автоматизацією, CLI, CI/CD.

> Якщо група менша за 6 осіб — суміщайте: QA + DevOps, або Architect + один Developer.

### Фіксація результату

PM записує на дошці або в GitHub Issue:

```text
Команда: [назва команди]
- Project Manager: [ім'я]
- Architect:       [ім'я]
- Developer:       [ім'я]
- Developer:       [ім'я]
- QA Engineer:     [ім'я]
- DevOps:          [ім'я]
```

> **Назву команди** вигадує сама команда — 3 хвилини максимум.

Кожен клонує репозиторій на свій комп'ютер:

```bash

cd <назва-папки>
```

**Перевірка**: ви бачите папку `Project_Khpi/project/` з файлами `README.md`, `01_Requirements/`, `02_Architecture/` — значить все правильно.

---

### Крок 2.4: Налаштування GitHub Project

PM виконує ці кроки в браузері:

1. Перейдіть до репозиторію на GitHub
2. Вкладка **Projects** → **New project**
3. Оберіть шаблон **Board** (kanban)
4. Назвіть проєкт: `Finance Assistant — [Назва команди]`
5. Додайте колонки:
   - `📋 Backlog`
   - `🚀 In Progress`
   - `👀 In Review`
   - `✅ Done`

Решта команди **приєднуються** до GitHub Project (PM запрошує через Settings проєкту).

## Блок 4: Перші GitHub Issues (10 хв)

### Що таке Issue

Issue = одиниця роботи. Має:

- **Назву** — дієслово + результат: «Налаштувати підключення до PostgreSQL»
- **Опис** — що потрібно зробити і як перевірити
- **Assignee** — хто відповідальний
- **Label** — `setup`, `feature`, `research`, `documentation`

### Задача PM на ці 10 хвилин

PM разом із Architect і командою створює **мінімум 3 Issues**:

```text
Issue #1: Налаштувати локальне dev-середовище
  Assignee: всі Developers
  Label: setup
  Опис: Встановити Node.js 20+, TypeScript, налаштувати локальний PostgreSQL.
        Результат: npm run build виконується без помилок.

Issue #2: Вивчити архітектуру Finance Skill
  Assignee: Architect
  Label: research
  Опис: Прочитати System_Architecture.md, запропонувати структуру папок
        для 10_Implementation/src/. Зафіксувати у першому ADR.

Issue #3: Написати User Stories для логування витрат
  Assignee: PM + QA
  Label: requirements
  Опис: На основі Business_Requirements.md написати 5 User Stories
        у форматі «Як [роль], я хочу [дія], щоб [результат]».
        Додати Acceptance Criteria до кожної.
```

Додайте Issues в колонку `📋 Backlog` на GitHub Project.
