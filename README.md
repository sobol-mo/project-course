# Навчальна дисципліна «Проєкт» — AI Assistant

> **Спеціальність**: Системний аналіз та управління, 3 курс, весна 2026
> **Обсяг**: 120 годин (16 год аудиторних + 104 год самостійної роботи)
> **Роль викладача**: Product Owner (визначає вимоги, приймає результати)

---

## Що ми будуємо?

Ми не навчаємося «на прикладах» — ми **реально будуємо** функціональність для живого AI-асистента, яким користується наш викладач щодня.

**Система**: Персональний AI-асистент у Telegram, що допомагає вести домашній бюджет у кількох валютах.

**Наша задача**: Реалізувати навичку (skill) для AI-агента — **Finance Skill**:

- Логування витрат та доходів через природну мову в Telegram
- Підтримка кількох валют: UAH, USD, EUR, USDT
- Аналітика: «Скільки витратив цього місяця?», «Покажи витрати по категоріях»
- Мультикористувацький режим (кілька членів домогосподарства)

> **Якщо встигнемо**: Health Skill — логування стану здоров'я, прийому ліків, аналіз.

---

## Як ми працюємо?

Дисципліна побудована за методологією **Scrum** з 2-тижневими спринтами.

**Аудиторна зустріч (2 год) = Scrum-церемонії**:

1. Sprint Review — демонстрація того, що зроблено за 2 тижні (30 хв)
2. Ретроспектива — що пішло добре / що покращити (20 хв)
3. Knowledge input — мінімальна теорія для наступного спринту (20 хв)
4. Sprint Planning — задачі на наступні 2 тижні (30 хв)

**Між зустрічами (~13 год на спринт)** — реальна командна розробка через GitHub:

- Pull Requests, Code Review, GitHub Issues
- Викладач як PO відповідає на питання в Issues/Comments, але не пише код

## Технічний стек

- **Агент**: ClawdBot (TypeScript / Node.js)
- **База даних**: PostgreSQL
- **Управління задачами**: GitHub Projects + GitHub Issues
- **Контроль версій**: Git + GitHub Flow (feature branches + PR)
- **CI/CD**: GitHub Actions
- **Тестування**: Jest (TypeScript)
