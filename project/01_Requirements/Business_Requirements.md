# Бізнес-вимоги: Finance Assistant

**Версія**: 1.0
**Дата**: 2026-03-10
**Статус**: Затверджено

---

## 1. Контекст і мета

### Проблема

Більшість людей не ведуть облік особистих фінансів — це нудно, вимагає відкривати окремий застосунок і заповнювати форми. В результаті вони не розуміють, куди йдуть гроші, і не можуть планувати витрати.

### Рішення

Telegram-бот з AI, якому можна писати природною мовою так само, як пишеш другу. Бот сам розпізнає суму, валюту і категорію — користувач просто фіксує факт.

### Ціль проєкту

Реалізувати **Finance Skill** — навичку AI-агента ClawdBot, яка:

1. Приймає повідомлення природною мовою
2. Зберігає фінансові операції у PostgreSQL
3. Надає аналітику на запит

## 2. Бізнес-цілі

| ID   | Ціль                                           | Показник успіху                                              |
| ---- | ---------------------------------------------- | ------------------------------------------------------------ |
| BC-1 | Знизити поріг входу в облік фінансів           | Запис нової операції займає менше 10 секунд                  |
| BC-2 | Підтримати кількох членів домогосподарства     | 2+ користувачі ведуть облік незалежно в одному боті          |
| BC-3 | Підтримати мультивалютні операції              | Система коректно зберігає UAH, USD, EUR, USDT без конвертації |
| BC-4 | Надати аналітику на запит                      | Звіт за будь-який місяць формується за 3 секунди             |

## 3. Користувачі системи

| Роль               | Опис                                                      |
| ------------------ | --------------------------------------------------------- |
| **Основний користувач** | Активно веде облік, налаштовує категорії           |
| **Другорядний користувач** | Веде облік своїх операцій, бачить спільні звіти |

## 4. Функціональні вимоги

### 4.1 Логування операцій

| ID          | Вимога                                                                 | Приклад вводу                          |
| ----------- | ---------------------------------------------------------------------- | -------------------------------------- |
| FR-FIN-001  | Записати витрату з сумою, валютою і описом                             | «Витратив 850 грн на продукти»         |
| FR-FIN-002  | Записати дохід з сумою, валютою і джерелом                             | «Отримав $500 фріланс»                 |
| FR-FIN-003  | Автоматично визначити валюту з тексту                                  | «грн» → UAH, «$» → USD, «євро» → EUR  |
| FR-FIN-004  | Автоматично визначити категорію за описом                              | «продукти» → «Їжа», «АТБ» → «Їжа»    |
| FR-FIN-005  | Підтвердити операцію, зачитавши збережені дані назад (Commit-Recall-Verify) | ✅ Записано: -850 ₴ · Їжа · сьогодні |
| FR-FIN-006  | Зберегти оригінальний текст користувача для аудиту                     | поле `raw_input`                       |
| FR-FIN-007  | Асоціювати операцію з користувачем (Telegram ID)                       | кожна операція має `user_id`           |

### 4.2 Підтримка валют

| ID          | Вимога                                                        |
| ----------- | ------------------------------------------------------------- |
| FR-FIN-010  | Підтримувати фіатні валюти: UAH, USD, EUR                     |
| FR-FIN-011  | Підтримувати криптовалюти: USDT, USDC                         |
| FR-FIN-012  | Зберігати код валюти явно у кожній транзакції                 |
| FR-FIN-013  | НЕ конвертувати валюти автоматично — зберігати в оригіналі    |
| FR-FIN-014  | Виводити суми у форматі валюти: «1 250 ₴», «$125.50»          |

### 4.3 Аналітика та запити

| ID          | Вимога                                                              | Приклад запиту                              |
| ----------- | ------------------------------------------------------------------- | ------------------------------------------- |
| FR-FIN-020  | Показати витрати за місяць з розбивкою по валютах                   | «Скільки витратив цього місяця?»            |
| FR-FIN-021  | Показати витрати за обраний місяць                                  | «Покажи витрати за квітень»                 |
| FR-FIN-022  | Показати витрати по категоріях за період                            | «Скільки витратив на їжу цього місяця?»     |
| FR-FIN-023  | Показати доходи за період                                           | «Скільки заробив цього кварталу?»           |
| FR-FIN-024  | Показати зведений звіт: доходи мінус витрати по валютах             | «Підсумок за травень»                       |

### 4.4 Мультикористувацький режим

| ID          | Вимога                                                                 |
| ----------- | ---------------------------------------------------------------------- |
| FR-FIN-030  | Кожен користувач веде облік незалежно (ізоляція даних)                 |
| FR-FIN-031  | За запитом виводити зведений звіт по всіх користувачах домогосподарства |
| FR-FIN-032  | «Мої витрати» → тільки поточний користувач; «Всі витрати» → всі        |

## 5. Нефункціональні вимоги

| ID    | Вимога                                                                 |
| ----- | ---------------------------------------------------------------------- |
| NFR-1 | Час відповіді бота на запис операції: менше 3 секунд                   |
| NFR-2 | Час формування аналітичного звіту: менше 5 секунд                      |
| NFR-3 | Дані зберігаються тільки в PostgreSQL (не в пам'яті бота)              |
| NFR-4 | Будь-яка операція має `raw_input` — оригінальний текст для аудиту      |
| NFR-5 | Код покритий тестами на рівні не менше 60% для бізнес-логіки           |

---

## 6. Обмеження та рішення

| Рішення                                    | Причина                                                         |
| ------------------------------------------ | --------------------------------------------------------------- |
| Без автоматичної конвертації валют          | Уникнути помилок курсу, відповідати вимогам обліку              |
| Категоризація через AI + правила, не тільки AI | Знизити кількість помилкових категорій                     |
| Multi-user тільки для фінансів             | Здоров'я та особисті дані — строго приватні (один користувач)   |
| PostgreSQL, без NoSQL                      | ACID-транзакції обов'язкові для фінансових даних                |

---

## 7. Поза межами цього проєкту

- Автоматичний імпорт з банківських виписок
- Конвертація валют за поточним курсом
- Бюджетування та ліміти витрат (може бути в наступній ітерації)
- Мобільний або веб-інтерфейс (тільки Telegram)
