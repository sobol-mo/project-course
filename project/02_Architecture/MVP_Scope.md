# MVP Scope: Finance Skill

**Версія**: 1.0
**Дата**: 2026-03-10
**Статус**: Активний

---

## Визначення MVP

MVP — мінімальний набір функціональності, якого достатньо, щоб реальний користувач міг вести облік фінансів через Telegram і отримувати корисну аналітику.

### Гіпотеза

> «Якщо записувати фінансові операції займає менше 10 секунд і не вимагає відкривати окремий застосунок — люди будуть це робити регулярно.»

---

## Критерії успіху MVP

| Критерій                                           | Метрика                                                |
| -------------------------------------------------- | ------------------------------------------------------ |
| Запис операції через природну мову                 | Від тексту до підтвердження ≤ 3 секунди                |
| Коректність розпізнавання валюти                   | 100% правильне визначення UAH / USD / EUR / USDT       |
| Зберігання даних у PostgreSQL                      | Кожна операція підтверджується зчитуванням з БД        |
| Аналітика за місяць формується правильно           | Звіт збігається з ручним підрахунком                   |
| Мінімум 2 користувачі ведуть облік незалежно       | Дані ізольовані, спільний звіт доступний               |

---

## Що входить у MVP

### Логування операцій (обов'язково)

- [ ] Запис витрат: сума + валюта + категорія + дата
- [ ] Запис доходів: сума + валюта + джерело + дата
- [ ] Авторозпізнавання валюти з тексту
- [ ] Авторозпізнавання категорії (AI + базові правила)
- [ ] Підтвердження операції з даними з БД (Commit-Recall-Verify)
- [ ] Зберігання оригінального тексту (`raw_input`)

### Аналітика (обов'язково)

- [ ] Витрати за поточний / обраний місяць по валютах
- [ ] Витрати по категоріях за обраний місяць
- [ ] Доходи за обраний місяць
- [ ] Зведений звіт: доходи − витрати

### Мультикористувацький режим (обов'язково)

- [ ] Ізоляція даних між користувачами
- [ ] Спільний звіт по домогосподарству

---

## Що НЕ входить у MVP

| Функція                                 | Причина виключення                        |
| --------------------------------------- | ----------------------------------------- |
| Автоматична конвертація валют           | Ризик помилок курсу, складність           |
| Бюджетування та ліміти категорій        | Наступна ітерація після MVP               |
| Імпорт банківських виписок (CSV/PDF)    | Потребує окремого парсера                 |
| Повторювані транзакції (регулярні платежі) | Складність планувальника              |
| Фото чеків (OCR)                        | Потребує інтеграції з Vision API          |
| Веб або мобільний інтерфейс            | Тільки Telegram у межах цього проєкту     |

## Покрокова доставка (спринти)

### Спринт 1 — Інфраструктура

**Мета**: ClawdBot може записати тестовий запис у БД і зчитати його назад.

Deliverables:
- ClawdBot підключений до PostgreSQL
- Тестова команда `!finance ping` → відповідь з БД

### Спринт 2 — Логування

**Мета**: Повний цикл логування операцій через природню мову.

Deliverables:
- «Витратив 850 грн на продукти» → запис у БД → підтвердження
- «Заробив $500 фріланс» → запис у БД → підтвердження
- Правильна валюта, категорія, `user_id`

### Спринт 3 — Аналітика

**Мета**: Користувач може отримати звіт за будь-який місяць.

Deliverables:
- «Скільки витратив цього місяця?» → зведений звіт по валютах
- «Покажи витрати по категоріях за квітень» → деталізований звіт
- «Підсумок за травень» → доходи − витрати
