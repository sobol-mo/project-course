## Мета дисципліни

Сформувати практичні навички роботи в **реальній команді** над реальним програмним проєктом, застосовуючи сучасні методології: Agile/Scrum, Git Flow, Code Review, CI/CD.

**Результат семестру**: Команда реалізує навички для живого AI-асистента. Обсяг визначається темпом команди.

## Ритм роботи

Дисципліна побудована за методологією **Scrum** з 2-тижневими спринтами.

```text
[Зустріч N]                               [Зустріч N+1]
    │                                           │
    ├─ Review: що зроблено за спринт           │
    ├─ Ретроспектива: що покращити             │
    ├─ Knowledge input: теорія за потребою      │
    └─ Planning: задачі на наступний спринт     │
                     │                          │
                     └── самостійна робота ─────┘
                         команди (~13 год)
```

Обсяг кожного спринту та вибір тем для knowledge input визначається **після ретроспективи** — виходячи з реального стану команди та беклогу.

Викладач між заняттями виконує роль **Product Owner**: відповідає на питання щодо функціональності через GitHub Issues/Comments, але **не пише код** і **не вирішує технічні питання** за студентів.

## Теми для опрацювання протягом семестру

Ці теми вводяться як knowledge input на зустрічах або як матеріали для самостійного вивчення — залежно від стану команди і беклогу.

### Вимоги та проєктування

- Agile-вимоги: User Stories, Acceptance Criteria, Definition of Done
- Architecture Decision Records (ADR): як фіксувати технічні рішення
- Читання і навігація по чужій кодовій базі

### Розробка та Git

- Git Flow: feature branches, pull requests, merge стратегії
- Культура Code Review: що перевіряти, як давати зворотний зв'язок
- TypeScript: типи, інтерфейси, async/await
- Підключення до PostgreSQL з Node.js (бібліотека `pg`)
- Архітектурний патерн: Commit-Recall-Verify

### Тестування

- Типи тестів: unit, integration, e2e — що і коли писати
- Jest: написання тестів для TypeScript
- Test Coverage: що таке 60% покриття і навіщо воно потрібне

### DevOps та автоматизація

- GitHub Actions: що таке CI/CD pipeline, як написати workflow
- Автоматичний запуск тестів на кожен PR
- Деплой навички в ClawdBot

### Презентація та ретроспектива

- Як демонструвати програмний продукт замовнику
- Формати ретроспективи: Start/Stop/Continue, 4Ls
- Технічна документація: що залишати після себе

## Definition of Done

### Finance Skill — мінімально прийнятний результат

- [ ] «Витратив X [валюта] на Y» → запис у `fin_transactions` з правильною категорією
- [ ] «Заробив X [валюта]» → запис із типом `income`
- [ ] «Скільки витратив цього місяця?» → структурована відповідь бота
- [ ] Покриття бізнес-логіки тестами >= 60%
- [ ] CI pipeline: тести проходять при кожному PR

### Health Skill — якщо встигнемо

- [ ] «Болить голова 7/10» → запис симптому
- [ ] «Прийняв магній 400 мг» → запис прийому ліків
